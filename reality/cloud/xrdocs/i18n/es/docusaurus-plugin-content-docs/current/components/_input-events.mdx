## Tipos

### Posición

| Propiedad | Tipo    | Descripción                                                 |
| --------- | ------- | ----------------------------------------------------------- |
| x         | entero  | La coordenada x en la pantalla, normalizada entre 0 y 1.    |
| y         | entero  | La coordenada y en la pantalla, normalizada entre 0 y 1.    |

### Evento UI

| Propiedad  | Tipo    | Descripción                                                 |
| ---------- | ------- | ----------------------------------------------------------- |
| x          | entero  | La coordenada x en la pantalla, normalizada entre 0 y 1.    |
| y          | entero  | La coordenada y en la pantalla, normalizada entre 0 y 1.    |

### Evento táctil

| Propiedad       | Tipo     | Descripción                                                                                                 |
| --------------- | -------- | ----------------------------------------------------------------------------------------------------------- |
| punteroId       | entero   | ID único para el puntero, proporcionado por el navegador.                                                   |
| posición        | Posición | Coordenadas de la posición tocada en la pantalla, normalizadas entre 0 y 1.                                 |
| worldPosition   | Vector3  | La posición en la que el touchedEvent golpeó en el mundo. _sólo disponible en `SCREEN_TOUCH_START`._        |
| objetivo        | eid      | eid si inicialmente tocó objeto                                                                             |
| iniciar         | Posición | Las coordenadas de posición donde se inició el evento táctil en la pantalla, normalizadas entre 0 y 1.      |
| cambiar         | Posición | Las coordenadas de posición donde se tocó desde el último cambio en la pantalla, normalizadas entre 0 y 1.  |

### GestoEvento

| Propiedad        | Tipo     | Descripción                                                                            |

| touchCount       | entero   | El número de puntos que contribuyen al gesto.                                          |

## Eventos

### UI_CLICK

Se emite cuando la pulsación (`UI_PRESSED`) y la liberación (`UI_RELEASED`) se producen en el mismo elemento de la interfaz de usuario. Este evento representa un gesto completo de clic o toque. Se envía al elemento en el que se solapan ambas interacciones y se utiliza normalmente para confirmar la intención del usuario, como activar un botón o desencadenar una acción.
El evento será enviado en el antepasado común más bajo del principio (presionado) y final (lanzado) eids. El movimiento del ratón no afecta a los eventos de clic.

La carga útil del evento es del tipo `Position`.

#### Ejemplo

<EventExample
  ns="ecs.input"
  event="UI_CLICK"
  callbackBody={`console.log('UI click: ', event.data.x, event.data.y)`}
/>

### UI_PRESSED

Se emite cuando el usuario inicia una interacción táctil o con el puntero hacia abajo en un elemento de la interfaz de usuario. Se envía únicamente al elemento exacto que se ha pulsado directamente y no se extiende a los elementos superiores. Este evento es útil para activar la retroalimentación visual inmediata o estados de interacción (como botones resaltados o animaciones) al inicio de la entrada del usuario.

La carga útil del evento es del tipo `TouchEvent`.

#### Ejemplo

<EventExample
  ns="ecs.input"
  event="UI_PRESSED"
  callbackBody={`console.log('UI pressed: ', event.data.position)`}
/>

### UI_RELEASED

Emitido cuando el puntero se levanta después de un `UI_PRESSED`. Siempre se envía al mismo elemento de la interfaz de usuario que se pulsó inicialmente, independientemente de dónde se suelte el puntero. Esto permite a los desarrolladores responder a la finalización de una interacción de pulsación, incluso si el puntero se alejó del objetivo original.

La carga útil del evento es del tipo `TouchEvent`.

#### Ejemplo

<EventExample
  ns="ecs.input"
  event="UI_RELEASED"
  callbackBody={`console.log('UI released: ', event.data.position)`}
/>

### UI_HOVER_START

Emite cuando el ratón comienza a pasar sobre un elemento de interfaz de usuario.

La carga útil del evento es del tipo `HoverEvent`.

#### Ejemplo

<EventExample
  ns="ecs.input"
  event="UI_HOVER_START"
  callbackBody={`console.log('UI hover start: ', event.data.x, event.data.y)`}
/>

### UI_HOVER_END

Emite cuando el ratón deja de pasar sobre un elemento de interfaz de usuario.

La carga útil del evento es del tipo `HoverEvent`.

#### Ejemplo

<EventExample
  ns="ecs.input"
  event="UI_HOVER_END"
  callbackBody={`console.log('UI hover end: ', event.data.x, event.data.y)`}
/>

:::note

- Varios puntos de contacto pueden estar activos simultáneamente.
- Sólo se reconocerá como activo un gesto táctil (simple o multitáctil) a la vez.
  :::

:::info
Si un evento táctil tiene un objetivo, se emitirá en ese objetivo y se propagará a sus elementos padre y finalmente al nivel global. Esto significa que un receptor táctil en un objeto padre capturará eventos de todos sus elementos hijos.
:::

### SCREEN_TOUCH_START

Se emite cuando el usuario toca o hace clic inicialmente en la pantalla o en el objeto de destino.

La carga útil del evento es del tipo `TouchEvent`.

#### Ejemplo

<EventExample
  ns="ecs.input"
  event="SCREEN_TOUCH_START"
  callbackBody={`console.log('Screen touch start: ', event.data.position)`}
/>

### SCREEN_TOUCH_MOVE

La carga útil del evento es del tipo `TouchEvent`.

Se emite cuando el usuario hace clic y arrastra o mueve el dedo por la pantalla.

#### Ejemplo

<EventExample
  ns="ecs.input"
  event="SCREEN_TOUCH_MOVE"
  callbackBody={`console.log('Screen touch move: ', event.data.position)`}
/>

### SCREEN_TOUCH_END

Se emite cuando el usuario deja de hacer clic o levanta el dedo de la pantalla.

La carga útil del evento es del tipo `TouchEvent`.

#### Ejemplo

<EventExample
  ns="ecs.input"
  event="SCREEN_TOUCH_END"
  callbackBody={`console.log('Screen touch end: ', event.data.position)`}
/>

:::info
Los eventos gestuales se emiten cuando el usuario hace un "gesto" en la pantalla del teléfono. Un gesto es cualquier acción que requiera varios dedos. Si el usuario comienza con una acción de "zoom" (2 dedos que se alejan el uno del otro) y luego añade otro dedo a la pantalla, el gesto de "zoom" terminará y comenzará uno nuevo con 3 dedos.
:::
