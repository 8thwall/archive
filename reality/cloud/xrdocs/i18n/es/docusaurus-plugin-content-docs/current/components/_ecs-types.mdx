Actualmente, **el almacenamiento de objetos o listas de tamaño dinámico no está soportado**. Estamos explorando activamente esta función y [**nos encantaría conocer sus casos de uso específicos**](https://forum.8thwall.com/).

Los siguientes tipos de datos son útiles para crear propiedades de esquema en un componente personalizado o referencias a un tipo específico.

| Tipo        | Descripción                        |
| ----------- | ---------------------------------- |
| ecs.eid     | Entidad de referencia              |
| ecs.f32     | Número en coma flotante de 32 bits |
| ecs.f64     | Número de coma flotante de 64 bits |
| ecs.i32     | Entero de 32 bits                  |
| ecs.ui8     | Entero sin signo de 8 bits         |
| ecs.ui32    | Entero de 32 bits sin signo        |
| ecs.cadena  | Cadena                             |
| ecs.boolean | Booleano                           |
