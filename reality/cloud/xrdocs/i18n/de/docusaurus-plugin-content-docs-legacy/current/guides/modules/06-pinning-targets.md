# Pinning-Ziele {#pinning-targets}

Wenn Sie ein Modul aus Ihrem eigenen Arbeitsbereich importieren, sehen Sie mehrere Optionen für das Anheftungsziel:

**Version**:
Wenn Sie ein "Version"-Anheftungsziel auswählen, können Sie Ihr importiertes Modul für Bugfix-Updates und kleinere Updates abonnieren oder automatische Modul-Updates deaktivieren. Diese Option wird nur angezeigt, wenn das Modul mindestens einmal bereitgestellt worden ist.

**Commit**:
Ermöglicht es Ihnen, das Modul an einen beliebigen Commit des Modulcodes anzuheften.

**Entwicklung**:
Ermöglicht die Verwendung von Modulcode, den Sie in Ihrem lokalen Client haben und der noch nicht veröffentlicht wurde. Wird für Entwicklung und Tests verwendet.
