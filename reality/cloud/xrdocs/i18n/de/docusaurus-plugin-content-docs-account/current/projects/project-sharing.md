# Gemeinsames Projekt

Die Projektfreigabe ermöglicht Mitgliedern eines anderen vertrauenswürdigen Arbeitsbereichs den Zugriff auf ein bestimmtes Projekt in Ihrem Arbeitsbereich
. Die Anzahl der Arbeitsbereiche, die Sie zum Zugriff auf Ihr Projekt einladen können, ist nicht begrenzt.

## Projektfreigabeberechtigungen {#permissions}

Mitglieder aus einem eingeladenen Arbeitsbereich **können**:

- Zugriff auf das gemeinsame Projekt von ihrem eigenen Arbeitsbereich aus.
- Code im Cloud-Editor bearbeiten und landen.
- Verwalten Sie Connected Domains (von 8th Wall gehostete Projekte) und/oder selbst gehostete Domains (selbst gehostete Projekte).
- Aktualisieren Sie die Meta-Informationen (Titel, Beschreibung, Titelbild usw.).
- Bildziele hinzufügen/bearbeiten.

Mitglieder aus einem eingeladenen Arbeitsbereich **können** nicht:

- Sie können Projekte in Ihrem Arbeitsbereich sehen, die noch nicht für sie freigegeben wurden.
- Kaufen oder verwalten Sie White-Label-Abonnements.
- Bearbeiten/Löschen Sie das Projekt.
- Verwalten Sie Freigaben und Berechtigungen.
- Grafik zur Nutzung und zu den jüngsten Trends anzeigen.

## Gemeinsames Projekt mit 8th Wall Support {#share-project-with-support}

1. Melden Sie sich als Benutzer mit den Berechtigungen `OWNER` oder `ADMIN` für den Arbeitsbereich an.
2. Stellen Sie sicher, dass Ihre letzten Änderungen [gelandet] sind (/studio/getting-started/build-land/)
3. Klicken Sie auf der Arbeitsbereichsseite auf das Menü "..." des gewünschten Projekts und dann auf **Freigeben**.
4. Klicken Sie auf **Einladung zur Unterstützung der 8. Wand**.

![](/images/account/project-sharing-support.gif)

## Projekt mit einem anderen Arbeitsbereich teilen {#share-project-with-another-workspace}

1. Melden Sie sich als Benutzer mit den Berechtigungen `OWNER` oder `ADMIN` für den Arbeitsbereich an.
2. Stellen Sie sicher, dass Ihre letzten Änderungen [gelandet] sind (/studio/getting-started/build-land/)
3. Klicken Sie auf der Arbeitsbereichsseite auf das Menü "..." des gewünschten Projekts und dann auf **Freigeben**:

![ShareProject](/images/account/share-project.jpg)

4. Geben Sie die **vollständige Arbeitsbereich-URL** (z. B. "https://www.8thwall.com/otherworkspace") oder den Arbeitsbereich
   **Kurzname** (z. B. "anderer Arbeitsbereich") ein und klicken Sie auf **Arbeitsbereich einladen**.

![ShareProjectUrl](/images/account/share-project-url.jpg)

5. Eine E-Mail-Einladung wird an die Benutzer `OWNER` und `ADMIN` vom eingeladenen Arbeitsbereich gesendet. Die Einladung
   muss innerhalb von sieben Tagen angenommen werden, sonst verfällt sie.

## Entfernen Sie den Zugriff auf einen Arbeitsbereich aus einem gemeinsamen Projekt {#remove-workspace-access-from-a-shared-project}

1. Melden Sie sich als Benutzer mit den Berechtigungen `OWNER` oder `ADMIN` für den Arbeitsbereich an.
2. Klicken Sie auf der Arbeitsbereichsseite auf das Menü "..." des gewünschten Projekts und dann auf **Freigeben**:

![ShareProject](/images/account/share-project.jpg)

3. Klicken Sie auf das Symbol **Mülleimer** neben dem Arbeitsbereich, den Sie vom Zugriff auf das Projekt ausschließen möchten.

![ProjectShareRemove](/images/account/share-project-remove.jpg)

4. Klicken Sie zur Bestätigung auf **Entfernen**.

## Zugriff auf ein von einem anderen Arbeitsbereich freigegebenes Projekt {#accessing-a-project-shared-by-another-workspace}

Wenn Sie eine Einladung zu Projekten angenommen haben, die anderen Arbeitsbereichen gehören, finden Sie diese unter der Registerkarte
**Externe Projekte** Ihres Arbeitsbereichs:

![ProjectSharedWithMe](/images/console-appkey-share-project-external-projects.jpg)
