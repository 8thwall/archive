# Eingabe

## Einführung

{frontMatter.description} Eingaben beziehen sich auf die verschiedenen Methoden, mit denen Benutzer mit der Anwendung interagieren. Derzeit werden die Eingabemethoden Touch, Tastatur, Maus und Gamepad unterstützt.

Siehe [world.input](/api/studio/world/input/) für api.

## Eingabe-Manager

In Studio gibt es Eingaben und Aktionen - der Eingabemanager richtet die Zuordnung zwischen ihnen ein. Eine Aktion ist eine Abstraktion über die Quelle der Eingabe(n), die sie erhält. Sie sind am nützlichsten für die Darstellung von Eingaben als "logische" Konzepte (z. B. "springen") und nicht als "physische" Eingaben (z. B. "Leertaste auf der Tastatur gedrückt").

![input-manager.png](/images/studio/input-manager.png)

### Aktionskarten

Sie können Action Maps über den Input Manager in den Raumeinstellungen verwalten (der Standardzustand des Inspektors, wenn kein Objekt ausgewählt ist).

#### Hinzufügen einer Aktionskarte

Action Map sind Sätze von Aktionen, die programmatisch ausgetauscht werden können. Um eine neue Aktionskarte zu erstellen, klicken Sie auf das Dropdown-Menü Aktionskarte und dann auf Neue Aktionskarte erstellen. Geben Sie dann den Namen der Aktionskarte ein und speichern Sie sie. Aktionskarten können umbenannt, dupliziert und gelöscht werden. Die Standard-Aktionskarte kann nur dupliziert werden.

![input-add-action-map.png](/images/studio/input-add-action-map.png)

:::info
Die aktuell ausgewählte Aktionskarte ist die Aktionskarte, die standardmäßig aktiv sein wird.
:::

#### Vorlagen für Aktionskarten

Wenn Sie Neue Aktionskarte erstellen wählen, werden mehrere Aktionskartenvorlagen angezeigt. Die Vorlagen Fly Controller und Orbit Controls werden für ihre jeweiligen Komponenten verwendet.

![input-action-map-templates.png](/images/studio/input-action-map-templates.png)

### Hinzufügen einer Aktion

Aktionen sind eine Abstraktionsschicht für Benutzereingaben. Um eine Aktion zur Aktionsübersicht hinzuzufügen, klicken Sie auf die Schaltfläche (+) Neue Aktion und geben Sie dann den Namen für Ihre Aktion ein.

![input-add-action.png](/images/studio/input-add-action.png)

### Bindung an eine Aktion

Bindungen sind die Art und Weise, wie Eingaben mit einer Aktion verknüpft werden können. Um eine Bindung hinzuzufügen, klicken Sie auf die Schaltfläche Bindung hinzufügen und wählen Sie in der Dropdown-Liste die Eingabe aus, die Sie an die Aktion binden möchten. Es gibt Bindungen für viele Arten von Eingaben, einschließlich Tastatur, Maus und Gamepad. Mehrere Eingabebindungen können auf eine einzige Aktion abgebildet werden. Um eine Bindung zu löschen, klicken Sie auf den Mülleimer neben der Bindung, die Sie löschen möchten.

![input-add-bindings.png](/images/studio/input-add-bindings.png)

### Aktionsmodifikatoren

Modifikatoren sind eine zusätzliche Eingabe, die Sie einer Bindung zuweisen können, um die Aktion zu aktivieren. Um eine Bindung mit einem Modifikator zu erstellen, klicken Sie auf die Schaltfläche Mit Modifikator. Der Modifikator wird mit dem ersten Dropdown-Menü ausgewählt und die Bindung mit dem zweiten Dropdown-Menü.

![input-add-modifier.png](/images/studio/input-add-modifier.png)

## Beispiele

### Erkennung einer Aktion im Tick

```ts
ecs.registerComponent({
  name: 'player-controller',
  tick: (world, component) => {
    if (world.input.getAction('jump')) {
      // Hier wird die Sprunglogik behandelt
    }
  },
})
```

### Erkennung einer Aktion in einem Zustandsautomaten

```ts
stateMachine: ({world, eid, schemaAttribute, dataAttribute}) => {
  ecs.defineState('default').initial().onTick(() => {
    if (world.input.getAction('jump'))) {
      // Hier wird die Sprunglogik gehandhabt
    }
  })
}
```

### Umschalten zwischen Aktionskarten

:::info
Wenn eine neue Aktionskarte eingestellt wird, wird die neue Karte erst beim nächsten Tick aktiviert.
:::

#### In der Zecke

```ts
tick: (world, component) => {
  if (world.input.getAction('menu')) {
    world.input.setActiveMap(world.input.getActiveMap() === 'player' ? 'menu' : 'player')
    return
  }
  if (world.input.getAction('jump')) {
    // Hier wird die Sprunglogik behandelt
  }
}
```

#### In einem Zustandsautomaten

```ts
stateMachine: ({world, eid, schemaAttribute, dataAttribute}) => {
  const switchMap = ecs.defineTrigger()

  ecs.defineState('default')
  .initial()
  .onEnter(() => {
    world.input.setActiveMap('player')
  })
  .onTick(() => {
    if (world.input.getAction('menu')) {
      switchMap.trigger()
      return
    }
    // Bewegungslogik hier behandeln
  }).onTrigger(switchMap, 'menu')

  ecs.defineState('menu').onEnter(() => {
        world.input.setActiveMap('menu')
  }).onTick(() => {
    if (world.input.getAction('menu')) {
      switchMap.trigger()
      return
    }
    // Hier wird die Menülogik behandelt
  }).onTrigger(switchMap, 'default')
}
```
