## Typen

### Position

| Eigentum | Typ      | Beschreibung                                                    |
| -------- | -------- | --------------------------------------------------------------- |
| x        | Ganzzahl | Die x-Koordinate auf dem Bildschirm, normiert zwischen 0 und 1. |
| y        | Ganzzahl | Die y-Koordinate auf dem Bildschirm, normiert zwischen 0 und 1. |

### UIHoverEvent

| Eigentum     | Typ      | Beschreibung                                                    |
| ------------ | -------- | --------------------------------------------------------------- |
| \-mal        | Ganzzahl | Die x-Koordinate auf dem Bildschirm, normiert zwischen 0 und 1. |
| y            | Ganzzahl | Die y-Koordinate auf dem Bildschirm, normiert zwischen 0 und 1. |
| ziele        | eid[]    | Das Ziel-UI-Element(e)                                          |

### TouchEvent

| Eigentum      | Typ       | Beschreibung                                                                                                              |
| ------------- | --------- | ------------------------------------------------------------------------------------------------------------------------- |
| pointerId     | Ganzzahl  | eindeutige ID für den Zeiger, die vom Browser bereitgestellt wird.                                                        |
| Position      | Position  | Koordinaten der berührten Position auf dem Bildschirm, normiert zwischen 0 und 1.                                         |
| weltPosition  | Vektor3   | Die Position, an der das touchedEvent in der Welt einschlug. nur verfügbar bei `SCREEN_TOUCH_START`.                      |
| Ziel          | eid       | eid wenn ursprünglich berührtes Objekt                                                                                    |
| Start         | Position  | Die Positionskoordinaten, an denen das Berührungsereignis auf dem Bildschirm begann, normiert zwischen 0 und 1.           |
| ändern        | Position  | Die Positionskoordinaten, an denen seit der letzten Änderung auf dem Bildschirm berührt wurde, normiert zwischen 0 und 1. |

### GestenEreignis

| Eigentum            | Art          | Beschreibung                                                                                |
| ------------------- | ------------ | ------------------------------------------------------------------------------------------- |
| touchCount          | Ganzzahl     | Die Anzahl der Punkte, die zur Geste beitragen.                                             |
| Position            | Position     | Koordinaten der berührten Position auf dem Bildschirm, normiert zwischen 0 und 1.           |
| startPosition       | Position     | Die Positionskoordinaten, an denen das Ereignis begann, normiert zwischen 0 und 1.          |
| PositionÄndern      | Position     | Die Positionskoordinaten seit der letzten Änderung, normiert zwischen 0 und 1.              |
| Verbreitung         | Schwimmer    | Die durchschnittliche Position zwischen den Zeigern vom Mittelpunkt aus.                    |
| startSpread         | Schwimmer    | Die erste Ausbreitung wurde in Angriff genommen.                                            |
| spreadChange        | Schwimmer    | Der Streuwert seit der letzten Änderung.                                                    |
| nextTouchCount      | Ganzzahl     | Am Ende die Anzahl der Zeiger, die an der folgenden Geste beteiligt sind                    |

### GamepadEvent

| Eigentum       | Typ                                                                   | Beschreibung          |
| -------------- | --------------------------------------------------------------------- | --------------------- |
| Gamepad        | [Gamepad](https://developer.mozilla.org/en-US/docs/Web/API/Gamepad)   | Das Gamepad-Objekt    |

## Veranstaltungen

### UI_CLICK

Wird ausgegeben, wenn sowohl das Drücken (`UI_PRESSED`) als auch das Loslassen (`UI_RELEASED`) auf demselben UI-Element erfolgt. Dieses Ereignis stellt eine vollständige Klick- oder Tippgeste dar. Sie wird an dem Element ausgelöst, an dem sich beide Interaktionen überschnitten haben, und dient in der Regel dazu, die Absicht des Benutzers zu bestätigen, z. B. eine Schaltfläche zu aktivieren oder eine Aktion auszulösen.
Das Ereignis wird auf dem kleinsten gemeinsamen Vorfahren des Starts (gedrückt) und Ende (freigegeben) eids versandt. Mausbewegungen haben keinen Einfluss auf Klickereignisse.

Die Nutzlast des Ereignisses ist vom Typ `Position`.

#### Beispiel

<EventExample
  ns="ecs.input"
  event="UI_CLICK"
  callbackBody={`console.log('UI click: ', event.data.x, event.data.y)`}
/>

### UI_PRESSED

Wird ausgegeben, wenn der Benutzer eine Berührung oder eine Interaktion mit dem Mauszeiger auf einem UI-Element auslöst. Sie wird nur auf genau dem Element ausgelöst, das direkt gedrückt wurde, und wirkt sich nicht auf übergeordnete Elemente aus. Dieses Ereignis ist nützlich, um sofortiges visuelles Feedback oder Interaktionszustände (z. B. hervorgehobene Schaltflächen oder Animationen) zu Beginn der Benutzereingabe auszulösen.

Die Nutzlast des Ereignisses ist vom Typ `TouchEvent`.

#### Beispiel

<EventExample
  ns="ecs.input"
  event="UI_PRESSED"
  callbackBody={`console.log('UI pressed: ', event.data.position)`}
/>

### UI_RELEASED

Wird ausgegeben, wenn der Zeiger nach einem `UI_PRESSED` angehoben wird. Sie wird immer auf demselben UI-Element ausgeführt, das ursprünglich gedrückt wurde, unabhängig davon, wo der Zeiger freigegeben wird. Dies ermöglicht es Entwicklern, auf den Abschluss einer Presseinteraktion zu reagieren, auch wenn sich der Zeiger vom ursprünglichen Ziel entfernt hat.

Die Nutzlast des Ereignisses ist vom Typ `TouchEvent`.

#### Beispiel

<EventExample
  ns="ecs.input"
  event="UI_RELEASED"
  callbackBody={`console.log('UI released: ', event.data.position)`}
/>

### UI_HOVER_START

Sendet, wenn die Maus über einem UI-Element fährt.

Die Nutzlast des Ereignisses ist vom Typ "HoverEvent".

#### Beispiel

<EventExample
  ns="ecs.input"
  event="UI_HOVER_START"
  callbackBody={`console.log('UI hover start: ', event.data.x, event.data.y)`}
/>

### UI_HOVER_END

Sendet, wenn die Maus nicht mehr über ein UI-Element fährt.

Die Nutzlast des Ereignisses ist vom Typ "HoverEvent".

#### Beispiel

<EventExample
  ns="ecs.input"
  event="UI_HOVER_END"
  callbackBody={`console.log('UI hover end: ', event.data.x, event.data.y)`}
/>

:::note

- Mehrere Touchpoints können gleichzeitig aktiv sein.
- Es wird jeweils nur eine Berührungsgeste (Einzel- oder Multitouch) als aktiv erkannt.
  :::

:::info
Wenn ein Berührungsereignis ein Ziel hat, wird es an diesem Ziel ausgelöst und auf seine übergeordneten Elemente und schließlich auf die globale Ebene übertragen. Das bedeutet, dass ein Touch-Listener auf einem übergeordneten Objekt Ereignisse von allen untergeordneten Elementen erfasst.
:::

### BILDSCHIRM_BERUEHRUNG_START

Wird ausgegeben, wenn der Benutzer den Bildschirm oder das Zielobjekt zum ersten Mal berührt oder anklickt.

Die Nutzlast des Ereignisses ist vom Typ `TouchEvent`.

#### Beispiel

<EventExample
  ns="ecs.input"
  event="SCREEN_TOUCH_START"
  callbackBody={`console.log('Screen touch start: ', event.data.position)`}
/>

### SCREEN_TOUCH_MOVE

Die Nutzlast des Ereignisses ist vom Typ `TouchEvent`.

Wird ausgestrahlt, wenn der Benutzer klickt und den Finger auf dem Bildschirm zieht oder bewegt.

#### Beispiel

<EventExample
  ns="ecs.input"
  event="SCREEN_TOUCH_MOVE"
  callbackBody={`console.log('Screen touch move: ', event.data.position)`}
/>

### BILDSCHIRM_BERUEHRUNG_ENDE

Wird ausgegeben, wenn der Benutzer aufhört zu klicken oder den Finger vom Bildschirm nimmt.

Die Nutzlast des Ereignisses ist vom Typ `TouchEvent`.

#### Beispiel

<EventExample
  ns="ecs.input"
  event="SCREEN_TOUCH_END"
  callbackBody={`console.log('Screen touch end: ', event.data.position)`}
/>

:::info
Gestenereignisse werden ausgelöst, wenn der Benutzer eine "Geste" auf dem Telefondisplay macht. Eine Geste ist jede Aktion, die mehrere Finger erfordert. Wenn der Benutzer mit einer "'Zoom"-Geste beginnt (2 Finger bewegen sich voneinander weg) und dann einen weiteren Finger auf den Bildschirm legt, endet die "Zoom"-Geste und eine neue mit 3 Fingern beginnt.
:::
