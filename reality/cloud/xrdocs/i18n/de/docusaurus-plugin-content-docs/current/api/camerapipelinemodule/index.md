# CameraPipelineModule

8th Wall Kameraanwendungen werden mit Hilfe eines Kamera-Pipeline-Modul-Frameworks erstellt. Anwendungen installieren Module, die dann das Verhalten der Anwendung zur Laufzeit steuern.

Unter [`XR8.addCameraPipelineModule()`](/api/xr8/addcamerapipelinemodule) finden Sie Einzelheiten zum Hinzufügen von Kamera-Pipeline-Modulen zu Ihrer Anwendung.

Ein Kamera-Pipeline-Modul-Objekt muss eine **.name** Zeichenfolge haben, die innerhalb der Anwendung eindeutig ist. Es sollte eine oder mehrere der folgenden Methoden für den Lebenszyklus der Kamera implementieren.  Diese Methoden werden an der entsprechenden Stelle in der Laufschleife ausgeführt.

Während der Hauptlaufzeit einer Anwendung durchläuft jedes Kamerabild den folgenden Zyklus:

`onBeforeRun` -> `onCameraStatusChange` (`requesting` -> `hasStream` -> `hasVideo` | `failed`) -> `onStart` -> `onAttach` -> `onProcessGpu` -> `onProcessCpu` -> `onUpdate` -> `onRender`

Kameramodule sollten eine oder mehrere der folgenden Methoden für den Lebenszyklus einer Kamera implementieren:

| Funktion                                                  | Beschreibung                                                                                                                                                                                                                                                                                                                                                                                                                  |
| --------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [onAppResourcesLoaded](onappresourcesloaded.md)           | Wird aufgerufen, wenn wir die mit einer App verbundenen Ressourcen vom Server erhalten haben.                                                                                                                                                                                                                                                                                                                                 |
| [onAttach](onattach.md)                                   | Wird aufgerufen, bevor ein Modul zum ersten Mal Rahmenaktualisierungen empfängt. Sie wird für Module aufgerufen, die entweder vor oder nach der Ausführung der Pipeline hinzugefügt wurden.                                                                                                                                                                                                                                   |
| [onBeforeRun](onbeforerun.md)                             | Wird unmittelbar nach [`XR8.run()`](/api/xr8/run) aufgerufen. Wenn irgendwelche Versprechen zurückgegeben werden, wartet XR auf alle Versprechen, bevor es weitergeht.                                                                                                                                                                                                                                                        |
| [onCameraStatusChange](oncamerastatuschange.md)           | Wird aufgerufen, wenn eine Änderung bei der Abfrage der Kamerarechte auftritt.                                                                                                                                                                                                                                                                                                                                                |
| [onCanvasSizeChange](oncanvassizechange.md)               | Wird aufgerufen, wenn sich die Größe der Leinwand ändert.                                                                                                                                                                                                                                                                                                                                                                     |
| [onDetach](ondetach.md)                                   | wird aufgerufen, nachdem ein Modul zum letzten Mal Rahmenaktualisierungen erhalten hat. Dies geschieht entweder, nachdem der Motor abgestellt oder das Modul manuell aus der Pipeline entfernt wurde, je nachdem, was zuerst eintritt.                                                                                                                                                                                        |
| [onDeviceOrientationChange](ondeviceorientationchange.md) | Wird aufgerufen, wenn das Gerät die Ausrichtung Querformat/Hochformat ändert.                                                                                                                                                                                                                                                                                                                                                 |
| [onException](onexception.md)                             | Wird aufgerufen, wenn ein Fehler in XR auftritt. Wird mit dem Fehlerobjekt aufgerufen.                                                                                                                                                                                                                                                                                                                                        |
| [onPaused](onpaused.md)                                   | Wird aufgerufen, wenn [`XR8.pause()`](/api/xr8/pause) aufgerufen wird.                                                                                                                                                                                                                                                                                                                                                        |
| [onProcessCpu](onprocesscpu.md)                           | Wird aufgerufen, um die Ergebnisse der GPU-Verarbeitung zu lesen und verwertbare Daten zurückzugeben.                                                                                                                                                                                                                                                                                                                         |
| [onProcessGpu](onprocessgpu.md)                           | Wird aufgerufen, um die GPU-Verarbeitung zu starten.                                                                                                                                                                                                                                                                                                                                                                          |
| [onRemove](onremove.md)                                   | wird aufgerufen, wenn ein Modul aus der Pipeline entfernt wird.                                                                                                                                                                                                                                                                                                                                                               |
| [onRender](onrender.md)                                   | Wird nach onUpdate aufgerufen. Dies ist der Zeitpunkt, an dem die Rendering-Engine alle WebGL-Zeichenbefehle ausgibt. Wenn eine Anwendung ihre eigene Laufschleife bereitstellt und sich auf [`XR8.runPreRender()`](/api/xr8/runprerender) und [`XR8.runPostRender()`](/api/xr8/runpostrender) verlässt, wird diese Methode nicht aufgerufen und das gesamte Rendering muss von der externen Laufschleife koordiniert werden. |
| [onResume](onresume.md)                                   | Wird aufgerufen, wenn [`XR8.resume()`](/api/xr8/resume) aufgerufen wird.                                                                                                                                                                                                                                                                                                                                                      |
| [onStart](onstart.md)                                     | Wird aufgerufen, wenn XR startet. Erster Callback nach [`XR8.run()`](/api/xr8/run) wird aufgerufen.                                                                                                                                                                                                                                                                                                                           |
| [onUpdate](onupdate.md)                                   | Wird aufgerufen, um die Szene vor dem Rendern zu aktualisieren. Die von den Modulen in [`onProcessGpu`](onprocessgpu.md) und [`onProcessCpu`](onprocesscpu.md) zurückgegebenen Daten werden als processGpu.modulename und processCpu.modulename vorliegen, wobei der Name durch module.name = "modulename" angegeben wird.                                                                                                    |
| [onVideoSizeChange](onvideosizechange.md)                 | Wird aufgerufen, wenn sich die Größe der Leinwand ändert.                                                                                                                                                                                                                                                                                                                                                                     |
| [requiredPermissions](requiredpermissions.md)             | Module können angeben, welche Browserfunktionen sie benötigen, für die möglicherweise Berechtigungsanfragen erforderlich sind. Diese können vom Framework verwendet werden, um die entsprechenden Berechtigungen anzufordern, wenn sie nicht vorhanden sind, oder um Komponenten zu erstellen, die die entsprechenden Berechtigungen anfordern, bevor XR ausgeführt wird.                                                     |

Hinweis: Kameramodule, die [`onProcessGpu`](onprocessgpu.md) oder [`onProcessCpu`](onprocesscpu.md) implementieren, können Daten an nachfolgende Stufen der Pipeline liefern. Dies geschieht über den Namen des Moduls.
