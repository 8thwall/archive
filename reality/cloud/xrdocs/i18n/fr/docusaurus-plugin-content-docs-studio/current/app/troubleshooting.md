# Dépannage

:::info[Public Bêta]
L'application de bureau **8th Wall est en version bêta publique** et ses fonctionnalités sont susceptibles d'être modifiées dans une prochaine version. Nous apprécions vos commentaires et nous avons un [forum d'assistance dédié aux utilisateurs de l'application bureautique bêta](https://forum.8thwall.com/c/desktop-beta/17)- merci de nous faire part de tous les problèmes que vous rencontrez ou de vos suggestions ici.
:::

## Limites

- Les répertoires de projets ne peuvent pas être hébergés sur un serveur externe, ils doivent utiliser le processus de construction et d'hébergement de 8th Wall.

## Problèmes connus

- Les modules ne sont pas pris en charge pour le moment.
- La modification des ensembles d'actifs n'est pas possible pour le moment.
- Il n'est pas possible de créer ou d'ouvrir des projets si le nom du projet correspond à un projet existant qui a été créé précédemment. Ce scénario peut se produire si vous êtes membre de plusieurs espaces de travail avec différents projets qui partagent le même nom.
- Les projets externes (partagés) ne peuvent pas être ouverts dans l'application de bureau. En savoir plus sur le partage externe de projets [ici](/account/projects/project-sharing/).
- Si vous changez de client alors que le simulateur est en cours d'exécution, le simulateur restera infiniment bloqué dans l'état "Initialisation". Vous pouvez fermer le projet et le rouvrir si vous entrez dans cet état.

## Recharger

Vous pouvez décharger et forcer le rechargement de l'application 8th Wall Desktop App à partir du menu **Visualiser** ou en utilisant les raccourcis clavier.
