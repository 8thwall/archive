# チーム

各ワークスペースには、それぞれ異なる権限を持つ 1 人以上のユーザを含むチームがあります。 ユーザーは複数のワークスペース チームに所属できます。 ユーザーは複数のワークスペース チームに所属できます。

他のユーザーをチームに追加して、ワークスペースのプロジェクトにアクセスできるようにします。 これにより、Web ARプロジェクトをチームとして共同で作成、管理、テスト、公開することができます。

## ユーザーを招待する {#invite-users}

1. ワークスペースを選択します。

![SelectWorkspace](/images/console-home-my-workspaces.jpg)

2. 左ナビゲーションの**チーム**をクリックします。

![SelectAccountNav](/images/console-workspace-nav-teams.jpg)

3. 招待するユーザーのメールアドレスを入力します。  複数のメールアドレスをカンマで区切って入力します。
4. ユーザーを招待する\*\*」をクリックする。

![ManagePlans](/images/console-teams-invite.jpg)

## ユーザー・ロール {#user-roles}

- OWNER
- ADMIN
- BILLMANAGER

## ユーザー・ハンドル {#user-handles}

ワークスペースの各ユーザーはハンドルを持っています。 ワークスペースのハンドルは、ユーザーによって既に取られているかカスタマイズされていない限り、ユーザーのプロファイルで定義されたユーザーハンドルと同じになります。

ハンドルはURLの一部として（"handle-client-workspace.dev.8thwall.app "の形式で）、8th Wall Cloud Editorで開発する際に新しい変更をプレビューするために使用されます。

例：tony-default-mycompany.dev.8thwall.app

:::danger

- ハンドルを変更する前に、クラウドエディターでの作業がすべて保存されていることを確認してください。
- ワークスペース内のプロジェクトに対する未着陸の変更はすべて破棄されます。
- ワークスペース内のプロジェクトで作成したクライアントはすべて削除されます。
  :::

#### ユーザーハンドルの変更 {#modify-user-handle}

1. ワークスペースを選択します。
2. 左ナビゲーションの**チーム**をクリックします。
3. 新しいハンドルを入力してください。
4. &#10004; をクリックして保存する。
5. 確認する
