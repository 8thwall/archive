# 認可されていないデバイス

問題ウェブアプリを表示しようとすると、「Device Not Authorized（デバイスが認証されていません）」というエラーメッセージが表示されます。

#### サファリ専用 {#safari-specific}

状況だ：

- プロジェクトを表示していると、「デバイスが認証されていません」というアラートが表示されます。
- [apps.8thwall.com/token](https://apps.8thwall.com/token) は正しい認可を示しています。

なぜこのようなことが起こるのか？

Safariにはインテリジェントトラッキング防止という機能があり、サードパーティのCookie（開発中にお客様のデバイスを認証するために使用するもの）をブロックすることができます。 ブロックされると、あなたのデバイスを確認することができません。 ブロックされると、あなたのデバイスを確認することができません。

修正の手順

1. クローズ・サファリ
2. 設定>Safari>クロスサイトトラッキングを防ぐ\\`で、インテリジェントトラッキング防止をオフにする。
3. 設定>Safari>詳細>ウェブサイトデータ>8thwall.com\\`で8th Wallのクッキーを消去する。
4. コンソールから再承認
5. プロジェクトをチェックする
6. 解決しない場合設定>Safari>Clear History and Website Data\\`ですべてのCookieを消去してください。
7. コンソールから再承認

#### それ以外の場合 {#otherwise}

トラブルシューティングの詳細については、#5以降の[Invalid App Key](/legacy/troubleshooting/invalid-app-key)の手順を参照してください。
