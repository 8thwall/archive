# 無効なアプリキー

問題自分のウェブアプリを表示しようとすると、\*\*"Invalid App Key "**または**"Domain Not Authorized "\*\*エラーメッセージが表示されます。

トラブルシューティングの手順

1. アプリのキーがソースコードに正しくペーストされているか確認してください。
2. Webアプリケーションに**https**で接続していることを確認してください。  これは、モバイルブラウザがカメラにアクセスするために必要です。  Webアプリケーションに**https**で接続していることを確認してください。  これは、モバイルブラウザがカメラにアクセスするために必要です。  これは、モバイルブラウザがカメラにアクセスするために必要です。
3. サポートされているブラウザを使用していることを確認してください。[Webブラウザの要件](/legacy/getting-started/requirements/#web-browser-requirements) を参照してください。
4. セルフホスティングの場合、デバイスが適切に認証されていることを確認する。  携帯電話で<https://apps.8thwall.com/token>にアクセスし、デバイスの認証ステータスを表示します。 Self Hosted Domains](/legacy/guides/projects/self-hosted-domains)を設定している場合は不要です。
5. 複数の Web Developer ワークスペースのメンバーである場合は、App Key と Dev Token が **同じワークスペース** のものであることを確認してください。
6. ウェブブラウザがプライベートブラウジングまたはインコグニートモードになっている場合は、プライベート/インコグニートモードを終了し、デバイスを再認証してから再度お試しください。
7. ウェブブラウザからウェブサイトのデータとクッキーを消去し、デバイスを再認証して、もう一度お試しください。
8. 有料のProプランまたはEnterpriseプランをご利用で、WebARエクスペリエンスに公開アクセスしようとしている場合は、[Self Hosted Domains](/legacy/guides/projects/self-hosted-domains)が適切に設定されていることを確認してください。
