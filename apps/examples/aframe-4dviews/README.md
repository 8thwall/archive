# A-Frame: 4DViews 

This example illustrates how to integrate a [`4Dviews`](https://www.4dviews.com) volumetric capture into an 8th Wall WebAR project using A-Frame.

![](https://media.giphy.com/media/Jk324c3bNjd9YJ3Hn5/giphy.gif)

### About 4DViews

4Dviews provides a full turn key solution to create volumetric video holograms through its HOLOSYS studios all around the world.
This example shows how to simply create an 8th Wall WebAR project using 4Dviews volumetric video files.
Follow these steps to create your own app, and contact [`4Dviews`](https://www.4dviews.com/contact-us) to get your volumetric videos.

### Project Overview

In **head.html**, we add these tags to get the 4DViews web player scripts:

    <script src="//dev.fourdviews.com/js/8thwall/model4D_Three.js" crossorigin="anonymous"></script>
    <script src="//dev.fourdviews.com/js/8thwall/web4dvResource.js" crossorigin="anonymous"></script>


**hologram-4ds.js** contains all the logic for the `holo4ds` component and `<hologram-4ds>` primitive. It is here you can further customize the hologram playback behavior.

In **body.html**, we add the ```<hologram-4ds>``` primitive to our ```<a-scene>``` which has a few important parameters:

- main-4ds: CDN link to the 4ds video file. It should be the "Mobile" version of 4ds file, classically generated by the 4DViews HOLOSYS studio.
- secondary-4ds: CDN link to the 4ds video file for devices not compatible with Mobile 4ds. It is mainly used on Desktop platforms and Safari iOS version < 14. It should be the "Desktop" version of 4ds file, classically generated by the 4DViews HOLOSYS studio.
- audio-4ds: the path to the audio file, if any. Keep it blank if there is no audio or if it is embedded in the 4ds file.
- size: starting size of hologram (default: 1.0)
- touch-target-size: size of touch target cylinder: 'height radius' (default: '1.5 0.5')
- touch-target-offset: offset of touch target cylinder: 'x z' (default: '0 0')
- touch-target-visible: if true, show touch target for debugging (default: false)

The 4D hologram is represented by the object **model4DS**.
Here is the list of the functions you can use on it :

- model4DS.**load**(showPlaceholder, playOnLoad, callback) -> method that will load (i.e. start downloading) the 4ds file data. 
 - Set showPlaceholder to true if you want to display the first frame during the time it's loading. 
 - Set playOnLoad to true if you want the sequence to play autmatically when it's been loaded. 
 - Add a callback function to be executed when the sequence has been loaded.
- model4DS.**pause**() -> pause the playback
- model4DS.**play**() -> start the playback
- model4DS.**update**() -> update the model (three.js mesh) for the current frame
- model4DS.**mute**() -> audio muting
- model4DS.**unmute**() -> audio unmuting
- model4DS.**destroy**() -> stop 4DS model, and clear loaded data
- model4DS.**keepChunksInCache**(boolean) -> keep the downloaded data in cache, to avoid a new download each time it loops (take care of the memory consumption)
- model4DS.**isLoaded**() -> return a boolean wether the sequence is loaded or not
- model4DS.**setWaitingGif**(url) -> set the displayed image during loading time
- model4DS.**model4D.setPosition**(Vec3) -> set the position of the 4DS model
- model4DS.**model4D.setRotation**(Vec3) -> set the rotation of the 4DS model

### About World Tracking

Built entirely using standards-compliant JavaScript and WebGL, 8th Wall’s Simultaneous Localization and Mapping (SLAM) engine is hyper-optimized for real-time AR on mobile browsers. Features include Six Degrees of Freedom (6-DoF), Lighting estimation, instant surface detection and responsive scale.

The Y position of the camera at start effectively determines the scale of virtual content on a surface (e.g. smaller y, bigger content). This can be reset at any time by calling [```recenter()```](https://www.8thwall.com/docs/web/#recenter).

The camera should NOT be at a height (Y) of zero. It must be set to a non-zero value.

### Optimizing for Metaversal Deployment

With R18, the all-new 8th Wall Engine features Metaversal Deployment, enabling you to create WebAR experiences once and deploy them to smartphones, tablets, computers and both AR and VR headsets. This project has a few platform-specific customizations:

In **body.html**, we use the default ```xrweb``` parameter ```"allowedDevices: mobile-and-headsets"``` on our```<a-scene>``` 
which ensures the project opens on mobile and headsets. 

In **body.html**, we add the ```"allowedDevices: any"``` parameter to our ```xrweb``` component in ```<a-scene>``` which ensures the project opens on all platforms, including desktop. By default, this is ```mobile-and-headsets```.
